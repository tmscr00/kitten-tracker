First time? Create a free project at supabase.com, then open the SQL editor and run this once to create the tables:

create table if not exists kitten_weights (
  id uuid primary key,
  date date not null,
  grams numeric not null,
  note text,
  created_at timestamptz default now()
);
create table if not exists kitten_sleeps (
  id uuid primary key,
  slept_at timestamptz not null,
  note text,
  created_at timestamptz default now()
);
-- Personal low-stakes data: allow the anon key to read/write.
alter table kitten_weights enable row level security;
alter table kitten_sleeps  enable row level security;
create policy "anon all w" on kitten_weights for all using (true) with check (true);
create policy "anon all s" on kitten_sleeps  for all using (true) with check (true);

⚠️ The anon key is public, so anyone who has it can read/write these two tables. That's fine for kitten data, but don't store anything sensitive there.
